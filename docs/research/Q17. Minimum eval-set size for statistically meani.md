# Q17. Minimum eval-set size for statistically meaningful comparison of DSP heuristic variants — how small can we keep the curated set while still detecting regressions?

There is no universal “magic number,” but for statistically meaningful detection of regressions between DSP‑heuristic variants you typically need **at least 20–50 curated instances**, and often 100+ if effects are small or variance is high.  The exact minimum depends on three things:[^1_1][^1_2][^1_3]

### 1. Effect size and variance

- If the new heuristic’s performance change is large (e.g., ≥5–10% on average), you can detect regressions with fewer instances (often 20–30).[^1_2][^1_3]
- If the regression is subtle (≤1–2%) or instance‑wise performance is noisy, you may need 100 or more to keep false‑negative and false‑positive rates low.[^1_3][^1_2]


### 2. How “statistically meaningful” is defined

- For a simple paired test (e.g., Wilcoxon or t‑test) at $\alpha = 0.05$ with 80% power, many empirical and simulation‑based studies suggest **no less than 20–30 samples** for nontrivial effect sizes.[^1_2][^1_3]
- When variance is high, recommendations for robust inference often push **minimums to 25–50 or more** to avoid unstable conclusions.[^1_1][^1_3]


### 3. Practical sweet spot for curated eval sets

- In practice, teams doing algorithm‑heuristic comparison often fix a **minimum of 50–100 expert‑curated instances** as a baseline, then extend the set if p‑values or confidence intervals remain wide.[^1_4][^1_2]
- If you can only keep the curated set small (say 20–40), you can:
    - explicitly state the **minimum detectable effect size** (e.g., “we can only reliably detect regressions ≥5%”);
    - use **cross‑validated or bootstrapped confidence intervals** to quantify uncertainty.[^1_3][^1_2]

In short: **you can often “keep it small” starting around 20–30, but 50+ is a safer default** if you want to reliably detect modest regressions in DSP‑heuristic variants.[^1_1][^1_2][^1_3]
